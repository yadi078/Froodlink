# FoodLink - Sistema de Comida Local

## Descripción
FoodLink es una plataforma que conecta cocineros locales con estudiantes y otras personas que buscan comida casera de calidad.

## Características Principales

### 1. Sistema de Usuarios
- Tres tipos de usuarios: Estudiante, Cocinero, Otro
- Registro e inicio de sesión
- Autenticación basada en localStorage (para demo)

### 2. Control de Permisos
- **Usuarios NO autenticados**: Pueden VER el menú pero NO pueden:
  - Realizar pedidos
  - Comentar en las comidas
  
- **Usuarios autenticados (Estudiante/Otro)**: Pueden:
  - Ver el menú completo
  - Realizar pedidos
  - Comentar en las comidas
  
- **Cocineros**: Pueden:
  - Ver el menú
  - Acceder al Panel de Cocinero
  - Agregar nuevas comidas
  - Editar sus comidas
  - Eliminar sus comidas
  - Ver cantidad disponible

### 3. Panel del Cocinero (CRUD)
- Crear nuevas comidas
- Ver listado de sus comidas
- Editar información de comidas
- Eliminar comidas que ya no están disponibles
- Gestionar cantidad disponible

## Configuración

### Base de Datos
1. Ejecutar el script: `scripts/create_database.sql`
2. Ejecutar el script: `scripts/seed_data.sql` (opcional, para datos de prueba)

### Backend API (Requiere implementación PHP)
El frontend está configurado para conectarse a: `http://localhost/foodlink/api`

Endpoints necesarios:
- `POST /login.php` - Autenticación de usuarios
- `POST /register.php` - Registro de nuevos usuarios
- `GET /get_comidas.php` - Obtener todas las comidas
- `GET /get_my_comidas.php` - Obtener comidas del cocinero
- `POST /add_comida.php` - Agregar nueva comida
- `POST /update_comida.php` - Actualizar comida
- `POST /delete_comida.php` - Eliminar comida
- `POST /create_pedido.php` - Crear pedido
- `POST /add_comentario.php` - Agregar comentario

### Modo Demo
La aplicación funciona en modo demo sin necesidad de backend:
- Usuarios de prueba están simulados
- Las operaciones se simulan localmente
- Datos de demostración están incluidos
